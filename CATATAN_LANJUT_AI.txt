GAYA KERJA YANG DIINGINKAN USER
- User biasa memanggil "bang" dan ingin jawaban bahasa Indonesia yang santai, jelas, dan langsung eksekusi.
- User ingin perubahan langsung dibuat, bukan hanya saran.
- User ingin desain terlihat profesional, responsif di desktop dan phone, tetapi tetap konsisten dengan tema aplikasi.
- Jangan membuat tampilan seperti landing page kosong. Aplikasi harus terasa seperti dashboard/web app yang siap dipakai.
- Kalau ada perubahan frontend, perhatikan screenshot dari user dan perbaiki bagian yang benar-benar dimaksud.
- Setelah selesai, beri ringkasan singkat: apa yang diubah, validasi apa yang sukses, commit hash, dan instruksi redeploy/hard refresh.

RAHASIA DAN .ENV
- File .env lokal berisi data sensitif. Jangan pernah menampilkan nilai asli dari .env di chat.
- .env sudah disiapkan untuk production/TiDB, tetapi tetap gitignored.
- AUTH_LOGIN_BYPASS di production harus false.
- Session dan cache diarahkan ke database untuk production.
- MYSQL_ATTR_SSL_CA memakai path relatif: storage/certs/tidb-ca.pem.
- Jika user minta env untuk Vercel, berikan template/nama key saja tanpa membocorkan nilai rahasia.

STACK PROJECT
- Laravel 12, Blade, PHP, MySQL/TiDB.
- Vite untuk asset frontend.
- CSS utama custom berada di public/css/kelas-catur.css.
- Layout aplikasi utama: resources/views/layouts/app.blade.php.
- Layout auth/login: resources/views/layouts/auth.blade.php dan resources/views/auth/login.blade.php.
- Dashboard: resources/views/dashboard/index.blade.php.
- Deploy target: Vercel dengan vercel-php.
- Database target: TiDB/MySQL.

FILE PENTING DEPLOY
- api/index.php: entrypoint Laravel untuk Vercel PHP.
- vercel.json: konfigurasi build/routing Vercel.
- .vercelignore: mengecualikan file yang tidak perlu.
- storage/certs/tidb-ca.pem: CA cert TiDB.
- config/database.php: sudah mendukung MYSQL_ATTR_SSL_CA relatif.
- .env.example: template env production.
- PANDUAN_DEPLOY_VERCEL.md: panduan deploy dan migrasi.
- package-lock.json: sudah ada.

- Dark mode tetap deep navy.
- Jangan pakai animasi berat. Buka/tutup menu cukup smooth ringan.
- Jangan bikin terlalu banyak spacing kosong di mobile.
- Semua device harus responsif.
- Sidebar mobile harus bisa ditutup via X, klik luar/backdrop, dan Escape.
- Sidebar mobile harus bisa scroll sampai menu paling bawah.
- Desktop collapsed sidebar harus proporsional dan tidak membuat konten terlalu mepet.

HAL YANG PERLU DIPERHATIKAN JIKA LANJUT EDIT CSS
- public/css/kelas-catur.css punya beberapa blok override historis. Aturan paling bawah biasanya dipakai untuk memastikan override menang.
- Jika menambah polish baru, lebih aman tambahkan blok kecil di bagian paling akhir CSS, lalu validasi.
- Jangan refactor besar CSS kecuali diminta, karena sudah banyak override untuk mengejar deploy cepat.
- Pastikan text tidak overlap dan tombol tidak keluar container di mobile.

HAL YANG PERLU DIPERHATIKAN JIKA LANJUT DEPLOY
- Setelah push ke main, Vercel biasanya auto redeploy.
- Jika tampilan belum berubah, minta user hard refresh Ctrl + F5 atau clear cache browser.
- Untuk migrasi production, gunakan command sesuai panduan: php artisan migrate --force, dijalankan di environment yang benar.
- Jangan menjalankan perintah destructive seperti reset hard.
